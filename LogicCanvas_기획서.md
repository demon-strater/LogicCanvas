# LogicCanvas 서비스 기획서

**버전**: v1.0  
**최종 수정일**: 2026년 2월 6일  
**문서 구분**: 서비스 상세 기획서

---

## 1. 서비스 개요

### 1.1 서비스 정의
LogicCanvas는 복잡한 문서들을 시각적 캔버스 위에 배치하고, AI를 활용하여 문서 간의 논리적 관계와 워크플로우를 자동으로 분석/시각화하는 **인지 매핑(Cognitive Mapping) 서비스**입니다.

### 1.2 핵심 가치
- 프로젝트의 전체 흐름을 한눈에 파악할 수 있는 **시각적 개요** 제공
- AI가 문서 내용을 분석하여 **자동으로 그룹핑 및 관계 연결** 수행
- FigJam/Miro와 유사한 **무한 캔버스** 기반의 자유로운 문서 배치
- 줌 인/아웃으로 **프로젝트 전체 조감도부터 개별 문서 상세**까지 탐색

### 1.3 대상 사용자
- 다수의 문서를 관리하며 프로젝트를 진행하는 기획자, 리서처, PM
- 문서 간의 관계를 시각적으로 파악하고 싶은 팀
- 리서치 → 기획 → 설계 → 실행 등 단계별 워크플로우가 있는 프로젝트 담당자

## 2. 정보 구조 (Information Architecture)

### 2.1 계층 구조
LogicCanvas는 3단계 계층 구조로 문서를 조직합니다:

```
대분류 (Major Group)
  └── 중분류 (Medium Group)
        └── 문서 (Document)
```

| 계층 | 명칭 | 역할 | 예시 |
|------|------|------|------|
| 1단계 | 대분류 (대그룹) | 프로젝트 워크플로우 단계 | 리서치, 기획, 설계, 실행, 분석, 보고 |
| 2단계 | 중분류 (중그룹) | 단계 내 세부 카테고리 | 데스크 리서치, 현장 조사, 인터뷰 |
| 3단계 | 문서 | 실제 콘텐츠 단위 | 감각과 심리적 지형도 리서치, 독특한 방향성의 리서치 |

### 2.2 데이터 모델

#### 문서 (Documents)
| 필드 | 타입 | 설명 |
|------|------|------|
| id | serial (PK) | 고유 식별자 |
| title | text | 문서 제목 |
| content | text | 문서 원문 내용 |
| summary | text (nullable) | 문서 요약 (AI 생성 또는 수동) |
| groupId | integer (nullable) | 소속 그룹 ID |
| x, y | integer | 캔버스 상 위치 좌표 |
| createdAt | timestamp | 생성일시 |
| updatedAt | timestamp | 수정일시 |

#### 문서 그룹 (Document Groups)
| 필드 | 타입 | 설명 |
|------|------|------|
| id | serial (PK) | 고유 식별자 |
| name | text | 그룹명 |
| description | text (nullable) | 그룹 설명 |
| parentId | integer (nullable) | 상위 그룹 ID (계층 구조) |
| x, y | integer | 캔버스 상 위치 좌표 |
| color | text | 그룹 색상 코드 (HEX) |
| monthStart | integer (nullable) | 타임라인 시작 월 (1-12) |
| monthEnd | integer (nullable) | 타임라인 종료 월 (1-12) |
| createdAt | timestamp | 생성일시 |

#### 문서 간 연결 (Document Edges)
| 필드 | 타입 | 설명 |
|------|------|------|
| id | serial (PK) | 고유 식별자 |
| sourceDocId | integer (FK) | 출발 문서 ID |
| targetDocId | integer (FK) | 도착 문서 ID |
| label | text (nullable) | 연결 설명 |
| edgeType | text | 연결 유형 (flow/depends/related/parent) |

#### 그룹 간 연결 (Group Edges)
| 필드 | 타입 | 설명 |
|------|------|------|
| id | serial (PK) | 고유 식별자 |
| sourceGroupId | integer (FK) | 출발 그룹 ID |
| targetGroupId | integer (FK) | 도착 그룹 ID |
| label | text (nullable) | 연결 설명 |
| edgeType | text | 연결 유형 (flow/depends/related) |

#### 노드 (Nodes) - 문서 내 개념 추출
| 필드 | 타입 | 설명 |
|------|------|------|
| id | serial (PK) | 고유 식별자 |
| documentId | integer (FK) | 소속 문서 ID |
| label | text | 개념 제목 (2-5단어) |
| content | text | 개념 상세 설명 |
| nodeType | text | 유형 (concept/claim/evidence/question) |
| x, y | integer | 노드 그래프 상 위치 |
| isTagged | boolean | 태그 여부 |
| tagNote | text (nullable) | 태그 메모 |

#### 태스크 (Tasks)
| 필드 | 타입 | 설명 |
|------|------|------|
| id | serial (PK) | 고유 식별자 |
| documentId | integer (FK) | 관련 문서 ID |
| nodeId | integer (FK, nullable) | 관련 노드 ID |
| title | text | 태스크 제목 |
| description | text (nullable) | 태스크 설명 |
| status | text | 상태 (pending/in_progress/completed) |
| priority | text | 우선순위 (low/medium/high) |

## 3. 화면 설계

### 3.1 전체 레이아웃

```
┌─────────────────────────────────────────────────┐
│  헤더 (Header) - h:56px                         │
│  [로고] LogicCanvas  문서 N개 · 그룹 N개  [다크모드]│
├─────────────────────────────────────────────────┤
│  타임라인 헤더 (Timeline) - h:48px               │
│  12월(2025) | 1월 | 2월 | 3월 | ... | 12월      │
├─────────────────────────────────────────────────┤
│                                                 │
│          무한 캔버스 영역                          │
│                                                 │
│   ┌──────────────┐     ┌──────────────┐         │
│   │ 리서치 (대그룹)│────→│ 기획 (대그룹) │         │
│   │  ┌─────────┐ │     │  ┌─────────┐ │         │
│   │  │중그룹    │ │     │  │중그룹    │ │         │
│   │  │ [문서1]  │ │     │  │ [문서3]  │ │         │
│   │  │ [문서2]  │ │     │  └─────────┘ │         │
│   │  └─────────┘ │     └──────────────┘         │
│   └──────────────┘                              │
│                                                 │
├───────┬─────────────────────────────────────────┤
│ [줌-] │ 21%  [줌+] | [리셋]                      │
│       │                    [재정렬] [자동정렬] [+] │
└───────┴─────────────────────────────────────────┘
```

### 3.2 헤더 (Header)
- **높이**: 56px (h-14)
- **구성요소**:
  - 좌: 로고 아이콘 (파란 그라데이션 배경의 FileText 아이콘) + 서비스명 "LogicCanvas"
  - 좌: 문서 개수 · 그룹 개수 표시
  - 우: 다크/라이트 모드 토글 버튼

### 3.3 타임라인 헤더 (Timeline Header)
- **높이**: 48px (h-12)
- **z-index**: 100 (항상 캔버스 위에 고정)
- **범위**: 2025년 12월 ~ 2026년 12월 (13개월)
- **기능**:
  - 월별 구분선으로 시간 축 제공
  - 문서 열람 시 해당 문서의 생성일에 **날짜 마커** 표시 (primary 색상 pill, FileText 아이콘, "N월 N일 | 문서제목" 텍스트)
  - 줌/팬에 따라 월 너비가 동적으로 변화
- **그리드 라인**: 각 월 경계에 세로 구분선, 주간 단위 점선 표시

### 3.4 캔버스 (Document Canvas)
- **유형**: 무한 캔버스 (Infinite Canvas)
- **배경**: 40px 간격 그리드 패턴 (매우 낮은 불투명도)
- **조작 방식**:

| 조작 | 방법 |
|------|------|
| 팬 (이동) | 스페이스바 + 드래그 또는 마우스 휠 클릭 드래그 |
| 줌 | 마우스 휠 스크롤 (0.05x ~ 2x 범위) |
| 단일 선택 | 요소 클릭 |
| 다중 선택 | Shift + 클릭 또는 빈 영역 드래그 (범위 선택) |
| 문서 열기 | 문서 박스 클릭 |
| 요소 이동 | 드래그 앤 드롭 (다중 선택 시 함께 이동) |
| 실행 취소 | Ctrl+Z (위치 이동 되돌리기) |

### 3.5 문서 박스 (Document Box)
- **크기**: 260px x 130px (최소 높이)
- **내용 구성**:
  - 아이콘 (FileText, primary 색상 배경)
  - 제목 (최대 2줄 말줄임)
  - 생성일 (한국어 형식: "2025년 1월 5일")
  - 요약 (최대 2줄, 150자 이내)
- **상태**:
  - 기본: 카드 배경, 테두리
  - 선택됨: primary 색상 테두리, 그림자
  - 드래그 중: 더 큰 그림자, grabbing 커서
  - 호버: primary/50 테두리, 그림자

### 3.6 그룹 박스 (Group Box)
- **패딩**: 30px (GROUP_PADDING)
- **헤더 높이**: 100px (GROUP_HEADER)
- **크기**: 내부 문서/하위 그룹의 바운딩 박스에 맞게 자동 계산
- **내용 구성**:
  - 폴더 아이콘 (그룹 색상)
  - 그룹명 (대그룹: text-sm, 중그룹: text-xs)
  - 하위 항목 수
  - 더보기 메뉴 (수정/삭제)
- **시각적 구분**:
  - 대그룹 (Top-Level): 투명 배경, 30% 불투명도 테두리
  - 중그룹 (Child): 반투명 배경, 80% 불투명도 테두리
- **색상**: 10가지 프리셋 색상 중 선택
  - Indigo (#6366f1), Violet (#8b5cf6), Pink (#ec4899), Red (#ef4444)
  - Orange (#f97316), Yellow (#eab308), Green (#22c55e), Teal (#14b8a6)
  - Sky (#0ea5e9), Gray (#6b7280)

### 3.7 연결선 (Connection Lines)

#### 문서 간 연결선
- **스타일**: FigJam 스타일의 부드러운 베지어 곡선
- **배경 할로**: 4px 두께 배경색 스트로크 (선 뒤에 깔림)
- **전경 선**: 1.5px 두께 색상 스트로크 (70% 불투명도)
- **화살표**: 오픈 셰브론 마커 (둥근 모서리, 8x6px)
- **곡률**: 거리 기반 자동 조절 (30~80px)

| 연결 유형 | 색상 | 의미 |
|-----------|------|------|
| flow | primary (파란색) | 순차적 워크플로우 |
| depends | destructive (빨간색) | 의존 관계 |
| parent | 녹색 (hsl 142, 76%, 36%) | 상위-하위 관계 |
| related | muted-foreground (회색, 점선) | 일반 연관 관계 |

#### 그룹 간 연결선
- **스트로크**: 2px 두께
- **화살표**: 10x8px 마커 (문서 간보다 약간 큼)
- **유형**: flow, depends만 지원

### 3.8 문서 상세 패널 (Document View Panel)
- **유형**: 우측 고정 사이드 패널 (기존 전체 화면 모달에서 변경)
- **위치**: 
  - top: calc(3.5rem + 48px + 12px) ≈ 116px
  - right: 16px
  - bottom: 16px
- **너비**: 520px (최대 100vw - 2rem)
- **z-index**: 200 (패널), 150 (반투명 배경)
- **반투명 배경**: 블러 처리된 검정 30% 배경 (헤더+타임라인 아래에서 시작)
- **구성**:
  - 상단: 문서 아이콘 + 제목 + 생성일시 + 삭제/닫기 버튼
  - 본문: "원문" 레이블 + 전체 문서 내용 (스크롤 가능)
- **닫기 방법**: 닫기 버튼 클릭, 배경 클릭, ESC 키
- **애니메이션**: 우측에서 슬라이드인 + 페이드인 (200ms)

### 3.9 하단 액션 바

#### 좌측 하단 - 줌 컨트롤
- 줌 아웃 버튼 (ZoomOut 아이콘)
- 현재 줌 레벨 표시 (%)
- 줌 인 버튼 (ZoomIn 아이콘)
- 구분선
- 리셋 버튼 (RotateCcw 아이콘) → 줌 1x + 중앙 정렬

#### 우측 하단 - 주요 액션
| 버튼 | 조건 | 기능 |
|------|------|------|
| 재정렬 | 문서 또는 그룹 1개 이상 | 기존 그룹 구조 유지하며 위치 재배치 |
| 자동 정렬 | 문서 2개 이상 | AI 워크플로우 분석 + 자동 그룹핑 |
| + (추가 메뉴) | 항상 표시 | 드롭다운: 새 문서 / 새 그룹 |

## 4. 기능 상세

### 4.1 문서 추가

#### 플로우
1. 우측 하단 "+" 버튼 클릭 → "새 문서" 선택
2. 문서 입력 모달 열림
3. 제목 입력
4. 내용 입력 방법 선택:
   - **텍스트 붙여넣기 탭**: 텍스트 영역에 직접 입력/붙여넣기
   - **파일 업로드 탭**: 텍스트 파일 선택 (파일명 → 제목 자동 입력)
5. "분석 시작" 버튼 클릭
6. AI가 문서 내용을 분석하여:
   - 개념(Concept), 주장(Claim), 근거(Evidence), 질문(Question) 추출
   - 요소 간 관계(supports, contradicts, implies, related) 매핑
7. 문서가 캔버스에 박스로 추가됨
8. 성공 토스트 메시지: "문서가 추가되었습니다"

#### AI 문서 분석 상세
- **사용 모델**: GPT-5.2 (OpenAI API)
- **추출 대상**: 5~15개 개념, 5~20개 관계
- **노드 유형**:
  - concept: 핵심 아이디어, 주제
  - claim: 주장, 논증
  - evidence: 사실, 데이터, 예시
  - question: 불확실성, 탐구 필요 영역
- **관계 유형**:
  - related: 일반적 연결
  - supports: 근거가 주장을 뒷받침
  - contradicts: 반대 또는 충돌
  - implies: 논리적 귀결

### 4.2 문서 열람
1. 캔버스의 문서 박스 클릭
2. 우측에 사이드 패널이 슬라이드 인으로 등장
3. 타임라인에 해당 문서의 생성일 마커 표시
4. 패널 내에서 원문 전체 스크롤 열람
5. 삭제 버튼(휴지통 아이콘)으로 문서 삭제 가능
6. 닫기: X 버튼, 배경 클릭, ESC 키

### 4.3 문서 삭제
1. 문서 상세 패널에서 삭제 아이콘 클릭
2. 문서 + 관련 노드 + 관련 엣지 자동 삭제 (CASCADE)
3. 성공 토스트: "문서가 삭제되었습니다"

### 4.4 그룹 관리

#### 그룹 생성
1. "+" 버튼 → "새 그룹" 선택
2. 그룹 입력 모달:
   - 그룹명 입력
   - 설명 입력 (선택)
   - 상위 그룹 선택 (선택, 드롭다운)
   - 색상 선택 (10가지 프리셋)
3. "그룹 만들기" 클릭
4. 성공 토스트: "그룹이 생성되었습니다"

#### 그룹 수정
1. 그룹 박스 헤더의 "..." 메뉴 → "수정"
2. 기존 값이 채워진 모달 열림
3. 수정 후 저장

#### 그룹 삭제
1. 그룹 박스 헤더의 "..." 메뉴 → "삭제"
2. 그룹 삭제 (소속 문서의 groupId는 null로 변경)
3. 성공 토스트: "그룹이 삭제되었습니다"

### 4.5 AI 자동 정렬 (워크플로우 분석)

#### 개요
2개 이상의 문서가 있을 때 AI가 전체 문서를 분석하여:
1. 워크플로우 단계별 대그룹 자동 생성 (리서치, 기획, 설계 등)
2. 각 단계 내 세부 중그룹 자동 생성
3. 문서를 적절한 그룹에 자동 배정
4. 문서 간 관계(flow, depends, related) 자동 생성
5. 그룹 간 워크플로우 연결선 자동 생성
6. 캔버스 상 위치 자동 배치

#### AI 분석 기준
- X축: 워크플로우 단계 순서 (리서치 좌측 → 보고 우측)
- Y축: 시간적 유사성 (같은 시기의 그룹은 비슷한 높이)
- 문서의 내용(content)과 생성일(createdAt)을 종합 분석
- 한국어 레이블 사용

#### 플로우
1. "자동 정렬" 버튼 클릭
2. "분석 중..." 로딩 상태 표시
3. 기존 문서 간 연결, 그룹, 그룹 간 연결 모두 초기화
4. AI가 새로운 구조 생성
5. 결과 반영: 문서 위치 + 그룹 생성 + 연결선 생성
6. 성공 토스트: "워크플로우 분석 완료 - N개 그룹 생성"

### 4.6 재정렬 (Relayout)

#### 개요
기존 그룹 구조를 유지한 채 위치만 재배치하는 기능.
AI가 아닌 알고리즘 기반으로 컴팩트하게 정렬합니다.

#### 정렬 규칙
- 대그룹: 좌→우 가로 배치 (500px 간격)
- 중그룹: 각 대그룹 내에서 세로 배치
- 문서: 각 중그룹 내에서 그리드 배치 (2열)
- 미분류 문서: 모든 그룹 우측에 별도 배치

### 4.7 캔버스 조작

#### 팬 (화면 이동)
- 스페이스바 홀드 시 커서가 grab → 드래그로 캔버스 이동
- 마우스 중앙 버튼(휠 클릭) 드래그로도 이동 가능
- 드래그 중 커서는 grabbing으로 변경

#### 줌 (확대/축소)
- 마우스 휠 스크롤로 줌 인/아웃
- 줌 범위: 5% ~ 200%
- 줌 스텝: 10%씩 변화
- 줌 중심점: 현재 마우스 위치 기준
- 좌측 하단 버튼으로도 줌 제어 가능
- 리셋 버튼: 1x 줌 + 콘텐츠 중앙 정렬

#### 선택 시스템
- **단일 선택**: 요소 클릭
- **토글 선택**: Shift + 클릭 (선택 추가/해제)
- **범위 선택**: 빈 영역에서 드래그하여 사각형 영역 내 요소 모두 선택
- **다중 이동**: 여러 요소 선택 후 하나를 드래그하면 모두 함께 이동
- **선택 해제**: 빈 영역 클릭

#### 실행 취소 (Undo)
- Ctrl+Z로 위치 이동 되돌리기
- 최대 50개 이력 저장 (문서/그룹 위치 변경)
- 토스트: "실행 취소됨"

### 4.8 다크 모드
- 헤더 우측의 토글 버튼으로 전환
- Sun/Moon 아이콘으로 현재 모드 표시
- localStorage에 설정 저장 (새로고침 시 유지)
- 모든 UI 요소가 다크/라이트 모드에 맞게 변환

## 7. 사용자 시나리오

### 시나리오 1: 프로젝트 문서 일괄 등록 및 자동 정리

> 기획자 A가 10개의 리서치/기획 문서를 하나씩 등록한 뒤, "자동 정렬"을 실행합니다.

1. "+" → "새 문서"로 문서 10개 순차 등록
2. 각 문서마다 AI가 개념 추출 (백그라운드)
3. "자동 정렬" 클릭
4. AI가 전체 문서를 분석하여:
   - "리서치" / "기획" / "설계" 3개 대그룹 자동 생성
   - 각 대그룹 아래 2~3개 중그룹 자동 생성
   - 각 문서를 적절한 그룹에 배정
   - 문서 간 flow/depends 관계 자동 연결
   - 그룹 간 워크플로우 화살표 생성
5. 캔버스에 정리된 프로젝트 맵이 표시됨
6. 줌 아웃하여 전체 프로젝트 조감도 확인
7. 필요시 문서/그룹을 드래그하여 위치 미세 조정

### 시나리오 2: 특정 문서 상세 확인

> 기획자 A가 캔버스에서 특정 문서의 원문을 확인합니다.

1. 캔버스에서 관심 문서 박스 클릭
2. 우측에 사이드 패널이 열림
3. 타임라인에 해당 문서의 생성일 마커 표시 ("2월 5일 | 독특한 방향성의...")
4. 패널 내에서 원문 전체 스크롤하며 확인
5. ESC 또는 닫기 버튼으로 패널 닫기
6. 다른 문서 클릭 시 패널 내용이 즉시 전환

### 시나리오 3: 수동 그룹 재구성

> 자동 정렬 결과가 마음에 들지 않아 수동으로 조정합니다.

1. "+" → "새 그룹"으로 원하는 그룹 수동 생성
2. 그룹 색상과 상위 그룹 지정
3. 문서들을 드래그하여 원하는 위치로 이동
4. 잘못 이동한 경우 Ctrl+Z로 되돌리기
5. "재정렬" 버튼으로 현재 그룹 구조 유지하며 깔끔하게 재배치

## 8. 디자인 가이드

### 8.1 색상 체계
- **Primary**: 파란색 계열 (브랜딩, 주요 액션, flow 연결선)
- **Destructive**: 빨간색 계열 (삭제, depends 연결선)
- **Muted**: 회색 계열 (보조 텍스트, related 연결선)
- **Card**: 카드/패널 배경색
- **Background**: 전체 배경색

### 8.2 텍스트 계층
- **기본**: 주요 텍스트 (foreground)
- **보조**: 부가 정보 (muted-foreground)
- **최소**: 날짜, 카운트 등 (text-[10px] muted-foreground)

### 8.3 간격 체계
- **소**: 4px (gap-1, p-1)
- **중**: 8~12px (gap-2~3, p-2~3)
- **대**: 16~24px (gap-4~6, p-4~6)

### 8.4 모서리 반경
- 문서 박스: rounded-md
- 그룹 박스: rounded-lg
- 버튼: rounded-md (기본), rounded-full (FAB)
- 패널: rounded-lg

### 8.5 그림자
- 기본: 없음 또는 매우 미세
- 호버: shadow-lg
- 드래그 중: shadow-xl
- 패널: shadow-2xl
